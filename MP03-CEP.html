Corporate Event Planner

Corporate Event Planner is a website that makes your event planning easy, efficient and successful. It helps you to organize and keep track of the events and collaborate with people on event planning.

The steps I had gone through for the product design process:

  • Discovery: User interview
  • Definition: Proto-persona
  • Ideation: Brainstorm for problem-solving, User Flows
  • Prototyping: Wireframe, Style guide/pattern library, InVision prototype

Tools I used: Sketch, InVision, Whimsical, Milanote

1. User Interview

Diane, she worked as a coordinator in InterContinental Hotels Group (IHG®) for 3 years. Part of her job responsibility is organizing conferences, team meetings, and team buildings for her team. She plans about 14 events every year.

3. Brainstorm for problem-solving

From MVP of the project:

  • Primary target users: The people who plan events for their teams and companies, such as HR, Coordinator, or Admin Assistance.
  • Basic functions: Event C.R.U.D.(create, read, update, delete)
  • Basic elements of an event: Name, Description, Date and time, Budget, Vendors, Shopping list, To-do-list.
  • Keyword: Event details, Keep track of events, People to coordinate
  • Key features: Organized event details, Easy to track, Collaborate with people.

Study how to plan events:Organized event details
1) Events are divided into current events and past events.
2) Sort by event type:
Conferences, Seminars, Meetings, Team Building Events, Business Dinners, Opening Ceremonies, Product Launches, Award Ceremonies, Charity Event, Other.
3) Break into different modules:
Since there are too many details, break into different modules enables the user to edit one module at a time. This also allows them to add the module they need.
Modules: Basic Information, Locations, Attendants, Agenda, Vendors, Budgets details, Caterings, Transportations, Entertainments, Supply list, and Others.
4) Default modules:
Set ‘Basic Information, Locations, Attendants, Agenda’ as four default modules. Other modules can be added through ‘add a new module’.
5) Event ratings:
Once the event has been launched, the user has an option to send ratings and comments request to all the attendants through the email. The system will collect the information and show in the past event.

4. User Flows

Based on the brainstorm for problem-solving, I created detailed user flows. It makes it easier to build the structure of the product for both designers and developers.

5. Wireframe

To visualize the user flows, I created wireframes for main pages. It allows us to focus on the function of the product and prioritizes the content of each page.

6. Style guide/pattern library

For high-fidelity design, a style guide is necessary for keeping the consistency of the design. Considering the target user, I have decided to make it modern and clean.
